# LLM-Storyteller: Supplementary Materials

Supplementary materials for "LLM-Storyteller: Child-Guided Narrative Co-Creation via Real-Time Adaptive Prompting and Visualization with an Embodied Robot" (Alali & Ezzini).

The session transcripts and outputs are released as produced by the system, with no editing or curation.

## Contents

```
compute_authorship.py    Computes the authorship indicators of Section 4.4
prompt_bank.json         The adaptive prompt bank (Listing 2)
sessions/                12 session logs, one Excel file per session
videos/                  Two recordings of the robot
```

## sessions/

One Excel file per session, 12 in total. Each has a `Turns` sheet, one row per dialogue turn:

| Column | Contents |
|---|---|
| `Turn #` | Turn index |
| `Child Text` | Child utterance as transcribed by the ASR |
| `Detected State` | Interaction state assigned by the classifier |
| `Prompt Type` | Prompt-bank entry selected |
| `Robot Response` | Follow-up question generated by the LLM |
| `Latency (s)` | Per-turn response latency |

```
session_21bcd1ac.xlsx    session_44164e6a.xlsx    session_7cb9af03.xlsx
session_2358e8ff.xlsx    session_5232fbe7.xlsx    session_ba2277d0.xlsx
session_3bc8538d.xlsx    session_53605343.xlsx    session_d04afae4.xlsx
session_3f588710.xlsx    session_58bd6a52.xlsx    session_f7b26ca7.xlsx
```

Files are named by session hash. S1 to S12 in the paper are these 12 logs ordered by mean per-turn response latency, ascending, which can be recomputed from the `Latency (s)` column.

## compute_authorship.py

```bash
pip install openpyxl
python compute_authorship.py sessions/
```

Prints LNO and question compliance per session and overall (Table 7). Expected totals:

```
  exceptions at/after turn 4   : 8
  exceptions before turn 4     : 6
```

## prompt_bank.json

The guiding questions, keyed by interaction state. One variant is drawn uniformly at random from the entry matching the detected state.
